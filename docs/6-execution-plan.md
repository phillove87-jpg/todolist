# TodoList 앱 개발 실행계획

## 1. 실행 원칙

- MVP 범위는 `회원가입`, `로그인`, `Todo 생성`, `Todo 목록 조회`, `Todo 완료 상태 변경`, `Todo 삭제`로 제한한다.
- 구현 기준 문서는 [1-prd.md](C:/_vibe/todolist/docs/1-prd.md), [2-erd.md](C:/_vibe/todolist/docs/2-erd.md), [3-api-spec.yaml](C:/_vibe/todolist/docs/3-api-spec.yaml), [5-wireframe.md](C:/_vibe/todolist/docs/5-wireframe.md), [5-1-style-guide.md](C:/_vibe/todolist/docs/5-1-style-guide.md) 이다.
- 1인 개발을 전제로, 각 Task는 다음 Task가 바로 이어질 수 있도록 산출물과 검증 기준을 남긴다.
- Supabase는 PostgreSQL 데이터베이스 용도로만 사용하고, 인증은 Express 백엔드의 JWT로 처리한다.
